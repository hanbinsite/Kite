## 一、阻断级问题（Critical — 10 项）

### C1. 构建失败：图标缺失 [✅ 确认]
- [ ] `tauri.conf.json:38-44` 引用 5 个图标，`icons/` 仅 1 个 `icon.ico`。`tauri build` 直接报错。
- 修复：设计 1024×1024 源图 → `pnpm tauri icon` 生成全套

### C2. AI Chat XSS [✅ 确认]
- [ ] `AiChatPanel.tsx:236` `dangerouslySetInnerHTML` 渲染 AI 和用户消息。`renderMarkdown` 正则不转义 HTML。
- 修复：escape HTML → DOMPurify 或换安全 markdown 库

### C3. Vault 密码学缺陷（3 合 1）[✅ 确认]
- [ ] **(3a)** `crypto.rs:84-87` 把 AES 主密钥 base64 写入 keyring（应只存密码，密钥在内存派生）
- [ ] **(3b)** `unlock_vault` 不验证密码正确性 → 输错密码"成功"，后续数据静默损坏
- [ ] **(3c)** `encrypt_vault_secret` 缺 name 路径校验 → `name="../../evil"` 写文件到 vault 外

### C4. Mock Server 绑定 0.0.0.0 [✅ 确认]
- [ ] `mock.rs:171` 监听所有网卡。应默认 `127.0.0.1`

### C5. Tauri 权限过宽 [✅ 确认]
- [ ] `capabilities/default.json` shell:allow-execute/allow-spawn/allow-stdin-write（→结合 C2 XSS 攻击链可达 RCE）
- [ ] `tauri.conf.json:28` CSP `connect-src *` 放开直连

### C6. Cmd+W 绕过未保存确认 [✅ 确认]
- [ ] `App.tsx:55` 直接 `closeTab; removeTabData`。**所有 Tab 关闭路径（按钮/快捷键/右键）均无 dirty 检查**
- 修复：抽取 `closeTabWithConfirm` 统一入口

### C7. WS/MQTT/SSE 连接泄漏 [✅ 确认]
- [ ] `removeConnection` 在 3 个 store 中定义但**全项目 0 次调用**。Tab 关闭不 disconnect → Rust 侧连接保持 + 前端 Record 永久增长

### C8. 协议面板无 UI 入口 [⚠️ 描述修正]
- [ ] Workbench 渲染已完备（5 个 Panel 可切换），但**无任何代码创建 protocol≠"http" 的 Tab**
- 严重程度：Critical（功能不可达）但非代码缺失问题

### C9. formdata 二进制文件上传路径遍历 [✅ 确认]（24-审计**未覆盖**）
- [ ] `http.rs:435` `std::fs::read(&param.value)` 无路径校验 → 可读任意文件

### C10. crypto.rs + engine.rs `unwrap()` [✅ 确认]（原定 Medium，核对后提级）
- [ ] `crypto.rs:148` `serde_json::to_string_pretty(&entry).unwrap()` — **生产路径 panic**
- [ ] `engine.rs:124` `Function::new(...).unwrap()` — **生产路径 panic**（隔离线程中，panic 表现为 timeout）
