# API Client 整体规划

## 1. 项目愿景
打造一款现代化、高性能、开发者友好的 API 客户端。融合 Postman 的功能完整性和 Bruno 的本地安全理念，深度集成 AI 辅助工作流，采用本地优先(Local-first)架构。

## 2. 目标用户
- 后端开发者：API 开发与调试
- 前端开发者：API 集成与测试
- 测试工程师：API 自动化测试
- DevOps工程师：API 监控与运维
- 技术管理者：API 治理与文档

## 3. 核心价值主张
- **现代化UI**：超越Postman的视觉体验，暗色模式优先
- **本地优先**：数据默认存储在本地，无需账户即可使用
- **Git友好**：集合以纯文本文件存储，天然支持版本控制
- **多协议统一**：HTTP/GraphQL/gRPC/WebSocket/MQTT/SSE 一站式管理
- **AI原生**：Agent Mode，自然语言驱动API操作
- **极速启动**：基于Tauri，冷启动<2秒

## 4. 产品定位
Postman 的功能 + Bruno 的本地安全 + Insomnia 的简洁。桌面端应用(优先) + Web端(辅助)，开源核心 + 高级功能订阅。

## 7. 技术选型

### 7.1 框架选择
| 选项 | 优势 | 劣势 | 决策 |
|------|------|------|------|
| Electron + React | 生态成熟 | 包体大(~150MB) | 备选 |
| Tauri + React | 包体小(~10MB)，性能高 | 生态较新 | **首选** |
| Tauri + Vue | 同上 | - | 备选 |
| Tauri + Svelte | 包体最小 | 生态新 | 观察 |

**决策**: Tauri 2.x + React 19 + TypeScript 5

### 7.3 架构模式
- **Clean Architecture**: 分层清晰，依赖倒置
- **Feature-Sliced Design**: 按功能模块组织代码
- **Local-first**: 本地优先，云同步可选
- **Rust-Backend-Networking**: 所有网络请求通过 Rust 后端发送，前端只负责 UI 和状态管理

|------|------|---------|

| 命令面板打开 | < 150ms (动画完成) | CSS 动画时长 |
| 集合树加载 (100 项) | < 100ms | `performance.now()` |
| 大 JSON 响应渲染 (1MB) | < 500ms | 前端计时 |

### 15.2 性能埋点代码

```typescript
// packages/core/src/performance/index.ts
export function measureAsync<T>(
  name: string,
  fn: () => Promise<T>
): Promise<T> {
  const start = performance.now();
  return fn().finally(() => {
    const duration = performance.now() - start;
    if (import.meta.env.DEV) {
      console.debug(`[perf] ${name}: ${duration.toFixed(1)}ms`);
    }
    // 生产环境可上报到 Sentry performance
  });
}

export function markPerformance(name: string) {
  performance.mark(`api-client:${name}`);
}

export function measurePerformance(startMark: string, endMark: string, name: string) {
  performance.measure(name, `api-client:${startMark}`, `api-client:${endMark}`);
  const entries = performance.getEntriesByName(name);
  if (entries.length > 0 && import.meta.env.DEV) {
    console.debug(`[perf] ${name}: ${entries[entries.length - 1].duration.toFixed(1)}ms`);
  }
}
```

### 15.3 内存监控

// 开发环境下监控内存使用
if (import.meta.env.DEV) {
  setInterval(() => {
    const mem = (performance as any).memory;
    if (mem) {
      const usedMB = (mem.usedJSHeapSize / 1024 / 1024).toFixed(1);
      const totalMB = (mem.totalJSHeapSize / 1024 / 1024).toFixed(1);
      if (parseFloat(usedMB) > 200) {
        console.warn(`[perf] High memory usage: ${usedMB}MB / ${totalMB}MB`);
      }
    }
  }, 30000);
