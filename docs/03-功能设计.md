## 2. HTTP 请求模块

### 2.1 模块职责
- 构建HTTP请求(支持所有HTTP方法)
- 管理请求参数(URL/Query/Headers/Body)
- 处理认证
- 发送请求并接收响应
- 支持HTTPS和代理

### 2.2 核心类型定义

```typescript
// HTTP 方法
type HttpMethod = 
  | 'GET' | 'POST' | 'PUT' | 'PATCH' | 'DELETE' 
  | 'HEAD' | 'OPTIONS' | 'TRACE' | 'CONNECT'
  | string; // 支持自定义方法

// URL 配置
interface UrlConfig {
  raw: string;                    // 原始URL
  protocol: string;               // http/https
  host: string[];                 // ['api', 'example', 'com']
  path: string[];                 // ['users', ':id']
  port: string;                   // '8080' | ''
  query: QueryParam[];            // 查询参数
  hash: string;                   // #section
  variable: VariableRef[];        // URL中的变量引用
}

// 查询参数
interface QueryParam {
  key: string;
  value: string;
  description?: string;
  disabled: boolean;
}

// 请求头
interface Header {
  key: string;
  value: string;
  description?: string;
  disabled: boolean;
  auto: boolean;                  // 自动添加的系统头
}

// 请求体
interface BodyConfig {
  mode: 'none' | 'formdata' | 'urlencoded' | 'raw' | 'binary' | 'graphql';
  formdata?: FormDataParam[];
  urlencoded?: QueryParam[];
  raw?: RawBody;
  binary?: string;                // 文件路径
  graphql?: GraphQLBody;
}

// Form Data
interface FormDataParam {
  key: string;
  value: string;
  type: 'text' | 'file';
  description?: string;
  disabled: boolean;
  contentType?: string;
}

// Raw Body
interface RawBody {
  language: 'text' | 'javascript' | 'json' | 'html' | 'xml' | 'yaml';
  content: string;
}

// GraphQL Body
interface GraphQLBody {
  query: string;
  variables: string;              // JSON string
  operationName?: string;
}

// 认证配置
interface AuthConfig {
  type: AuthType;
  config: Record<string, any>;
}

type AuthType = 
  | 'none' 
  | 'apikey' 
  | 'bearer' 
  | 'basic' 
  | 'jwt' 
  | 'oauth1' 
  | 'oauth2' 
  | 'awsv4' 
  | 'hawk' 
  | 'ntlm' 
  | 'digest';

// API Key Auth
interface ApiKeyAuth {
  key: string;
  value: string;
  addTo: 'header' | 'query';
}

// Bearer Auth
interface BearerAuth {
  token: string;
}

// Basic Auth
interface BasicAuth {
  username: string;
  password: string;
}

// JWT Auth
interface JWTAuth {
  payload: string;                // JSON string
  secret: string;
  algorithm: 'HS256' | 'HS384' | 'HS512' | 'RS256' | 'RS384' | 'RS512' | 'ES256' | 'ES384' | 'ES512' | 'PS256' | 'PS384' | 'PS512';
  addTo: 'header' | 'query';
  headerPrefix: string;
}

// OAuth 1.0
interface OAuth1Auth {
  consumerKey: string;
  consumerSecret: string;
  token: string;
  tokenSecret: string;
  signatureMethod: 'HMAC-SHA1' | 'HMAC-SHA256' | 'RSA-SHA1' | 'RSA-SHA256';
  version: '1.0';
  realm?: string;
}

// OAuth 2.0
interface OAuth2Auth {
  grantType: 'authorization_code' | 'client_credentials' | 'password' | 'implicit';
  authUrl?: string;
  accessTokenUrl?: string;
  clientId?: string;
  clientSecret?: string;
  scope?: string;
  state?: string;
  redirectUri?: string;
  pkce?: boolean;
  accessToken?: string;
  refreshToken?: string;
  expiresIn?: number;
}

// AWS Signature v4
interface AWSV4Auth {
  accessKey: string;
  secretKey: string;
  sessionToken?: string;
  service: string;
  region: string;
}

// 请求设置
interface RequestSettings {
  timeout: number;
  maxRedirects: number;
  followRedirects: boolean;
  skipSSLVerification: boolean;
  proxy?: ProxyConfig;
  keepAlive: boolean;
  sendCookies: boolean;
  storeCookies: boolean;
}

// 代理配置
interface ProxyConfig {
  proxyType: 'http' | 'socks5' | 'system';
  url: string;
  auth?: ProxyAuth;
  bypass: string[];
}

interface ProxyAuth {
  username: string;
  password: string;
}

// 完整请求对象
interface HttpRequest {
  id: string;
  name: string;
  description?: string;
  method: HttpMethod;
  url: UrlConfig;
  headers: Header[];
  params: QueryParam[];
  body: BodyConfig;
  auth: AuthConfig;
  preRequestScript?: string;
  postResponseScript?: string;
  settings: RequestSettings;
}
```

## 3. 响应模块

### 3.1 响应数据结构

```typescript
interface HttpResponse {
  id: string;
  requestId: string;
  status: number;                    // 200, 404, 500
  statusText: string;                // OK, Not Found, Internal Server Error
  headers: ResponseHeader[];
  body: string | ArrayBuffer;
  bodySize: number;                  // bytes
  contentType: string;               // application/json
  encoding: string;                  // utf-8, gzip
  time: number;                      // ms
  timeline: ResponseTimeline;
  cookies: Cookie[];
  testResults: TestResult[];
}

interface ResponseHeader {
  key: string;
  value: string;
  description?: string;
}

interface ResponseTimeline {
  total: number;
  dns: number;                       // DNS lookup
  tcp: number;                       // TCP connection
  tls: number;                       // TLS handshake
  firstByte: number;                 // Time to first byte
  contentDownload: number;           // Content download
}

interface Cookie {
  name: string;
  value: string;
  domain: string;
  path: string;
  expires: string;
  httpOnly: boolean;
  secure: boolean;
  sameSite: 'Strict' | 'Lax' | 'None';
}

interface TestResult {
  name: string;
  passed: boolean;
  error?: string;
  duration: number;                  // ms
}
```

## 4. 集合管理模块

### 4.1 数据模型

```typescript
// 集合
interface Collection {
  id: string;
  name: string;
  description?: string;
  version?: string;                  // 1.0.0
  variables: Variable[];
  auth?: AuthConfig;                 // 继承认证
  preRequestScript?: string;
  postResponseScript?: string;
  folders: Folder[];
  requests: Request[];
  events: EventConfig[];             // 集合级别的事件
  metadata: CollectionMetadata;
  createdAt: string;
  updatedAt: string;
}

interface CollectionMetadata {
  id: string;
  name: string;
  schema?: string;                   // collection schema URL
  exportedAt?: string;
  exporter?: string;
}

// 文件夹
interface Folder {
  id: string;
  name: string;
  description?: string;
  variables: Variable[];
  auth?: AuthConfig;
  preRequestScript?: string;
  postResponseScript?: string;
  folders: Folder[];                 // 递归嵌套
  requests: Request[];
  events: EventConfig[];
}

interface EventConfig {
  listen: 'prerequest' | 'test';
  script: ScriptConfig;
}

interface ScriptConfig {
  id: string;
  type: 'text/javascript' | 'application/javascript';
  exec: string[];
  src?: string;
}
```

### 4.2 存储格式

集合以文件系统方式存储:

```
collections/
└── my-api/
    ├── collection.json          # 集合元数据
    ├── README.md                # 集合文档
    ├── environments/
    │   └── development.json
    └── requests/
        ├── users/
        │   ├── list.req.json    # GET /users
        │   ├── create.req.json  # POST /users
        │   └── _folder.json     # 文件夹定义
        └── orders/
            └── get.req.json     # GET /orders/:id
```

**请求文件格式** (`.req.json`):
```json
{
  "id": "req_abc123",
  "name": "Get Users",
  "description": "Retrieve all users",
  "method": "GET",
  "url": "https://api.example.com/users",
  "headers": [],
  "params": [],
  "body": { "mode": "none" },
  "auth": { "type": "bearer", "config": { "token": "{{token}}" } }
}
```

### 4.3 集合操作

| 操作 | 描述 | 权限 |
|------|------|------|
| 创建集合 | 新建集合/文件夹 | 读写 |

| 复制请求 | 复制请求到其他集合 | 读写 |

| 同步集合 | 与云端同步 | 读写 |

## 5. 环境与变量模块

### 5.1 变量类型

```typescript
// 变量
interface Variable {
  id: string;
  key: string;
  value: string;
  type: 'string' | 'number' | 'boolean' | 'secret';
  description?: string;
  disabled: boolean;
}

// 变量作用域
type VariableScope = 'global' | 'collection' | 'environment' | 'data' | 'local';

// 动态变量类型
type DynamicVariable = 
  | '$guid'           // UUID v4
  | '$timestamp'      // Unix timestamp
  | '$isoTimestamp'   // ISO 8601
  | '$randomInt'      // 0-1000
  | '$randomFullName'
  | '$randomEmail'
  | '$randomUuid'
  | '$date';
```

### 5.2 变量解析顺序

优先级从低到高:
1. **Global** - 全局变量
2. **Collection** - 集合变量
3. **Environment** - 环境变量
4. **Data** - 数据文件变量 (CSV/JSON)
5. **Local** - 请求本地变量

```
{{variable}}
    │
    ▼
┌─────────────────────────────────────┐
│ 1. 检查 Local 变量                  │
│ 2. 检查 Data 变量                   │
│ 3. 检查 Environment 变量            │
│ 4. 检查 Collection 变量             │
│ 5. 检查 Global 变量                 │
│ 6. 检查 Vault (secret)              │
│ 7. 检查动态变量                     │
│ 8. 未找到 → 保持原样 {{variable}}   │
└─────────────────────────────────────┘
```

## 23. Computed Variables（计算变量）

### 23.1 概述

Computed Variables 是基于其他变量动态计算其值的变量。它们不是静态键值对，而是表达式。

### 23.2 数据模型

```typescript
interface ComputedVariable {
  key: string;
  expression: string;     // 如 "{{base_url}}/v1"
  resolvedValue?: string;  // 解析后的值
  error?: string;          // 解析错误信息
}

interface Environment {
  // ... 已有字段
  variables: EnvironmentVariable[];
  computed: ComputedVariable[];
}
```

### 23.3 解析规则

1. 从左到右解析表达式中的 `{{variable}}` 引用
2. 引用按作用域优先级解析（Local > Data > Env > Collection > Global > Vault > Dynamic）
3. 支持嵌套引用，最多 5 层递归（防止循环引用）
4. 循环引用检测：解析过程中维护 visited set，重复引用返回错误
5. 解析失败的变量（未定义或循环引用）保持 `{{key}}` 原样

### 23.4 前端展示

在环境编辑器的 "Computed" 区域展示（详见 07-核心页面视觉规范.md §5.5）：

```
┌── Computed ──────────────────────────────┐
│  full_url  = {{base_url}}/v1            │
│  → https://api.dev/v1                   │  ✓ 解析成功
│                                           │
│  broken   = {{undefined_var}}/path       │
│  → ⚠ Variable 'undefined_var' not found │  ✗ 解析失败
└───────────────────────────────────────────┘
```

### 23.5 Rust 端解析

```rust
pub fn resolve_computed_variables(
    expression: &str,
    variables: &HashMap<String, String>,
    max_depth: usize,
) -> Result<String, VariableError> {
    let mut result = expression.to_string();
    let mut depth = 0;
    let mut visited = HashSet::new();

    while result.contains("{{") && depth < max_depth {
        let new_result = replace_variables(&result, variables, &mut visited)?;
        if new_result == result {
            break; // 无法再替换
        }
        result = new_result;
        depth += 1;
    }

    Ok(result)
}
```
