# API Client UI 设计规范 v2

## 1. 设计理念

### 1.1 设计原则

- **克制即力量**：每个像素都有存在的理由，没有多余的装饰
- **命令面板为核心**：⌘K 是一切操作的入口，而非菜单和按钮
- **上下文优先**：界面元素按当前任务上下文出现和消失，不常驻
- **暗色即品牌**：暗色不是主题选项，是产品的视觉身份
- **动效即反馈**：每个状态变化都有可感知的动画反馈

### 1.2 设计灵感

- **Linear** — 极简、克制、命令面板驱动
- **Arc** — 侧边栏即导航、URL 栏合并工作区
- **Warp** — AI 原生、块状编辑、现代终端体验
- **Figma** — 面板按需展开、属性检查器模式
- **Raycast** — 命令面板即一切、扩展生态

### 1.3 反模式（绝对不做）

- ❌ Postman 式的五面板同屏
- ❌ 永远占据空间的右侧面板
- ❌ 塞满按钮的 Header
- ❌ 28px 高的 Footer 常驻显示
- ❌ 从 GitHub 照搬的色板
- ❌ 没有品牌识别度的"安全"设计

### 3.2 URL 栏（替代 Header）

URL 栏是唯一的"顶部导航"，它既是地址栏也是操作栏：

```
┌──────────────────────────────────────────────────────────────────┐
│ [≡] [←] [→]  [GET ▼]  [ api.example.com/users          ]  [▶] │
│                       [ Dev ▼ ]  [⌘K]                 [⚙ ▼]  │
└──────────────────────────────────────────────────────────────────┘
```

**元素详解**：

| 元素 | 宽度 | 说明 |
|------|------|------|
| ≡ 侧边栏切换 | 36px | 切换侧边栏显隐 |
| ← → 导航 | 36px×2 | 请求历史导航 |
| 方法选择器 | 自适应 | 彩色圆角标签，颜色随方法变化 |
| URL 输入框 | flex-1 | 灰色背景，变量 {{}} 蓝色高亮 |
| ▶ 发送按钮 | 40px | 品牌色，发送时脉冲动画 |
| Dev 环境选择器 | 自适应 | 药丸形，颜色随环境变化 |
| ⌘K 搜索入口 | 36px | 点击打开命令面板 |
| ⚙ 菜单 | 36px | 下拉：设置/主题/关于/退出 |

### 3.3 侧边栏

**精简为 3 个区域，不用 Tab 切换**：

```
┌──────────────────────┐
│ [⌘K] 搜索...   [+]   │  ← 搜索栏 + 新建按钮
├──────────────────────┤
│                      │
│  COLLECTIONS         │  ← 区域标题（可折叠）
│  ▸ My API            │
│    ├ GET /users      │
│    └ POST /users     │
│  ▸ Another API       │
│                      │
├──────────────────────┤
│                      │
│  HISTORY             │
│  Today               │
│    GET /users  2m ago│
│  Yesterday           │
│    POST /orders      │
│                      │
├──────────────────────┤
│                      │
│  ENVIRONMENTS        │
│  ● Development       │  ← 当前环境有圆点标记
│  ○ Staging           │
│  ○ Production        │
│                      │
└──────────────────────┘
```

**关键设计**：
- 无 Tab，直接纵向排列三个区域
- 每个区域可折叠/展开
- 集合区域占最大空间，History 和 Envs 可折叠
- 当前环境用品牌色圆点标记
- 底部无 Footer，环境信息在侧边栏内

响应状态信息以**内联药丸标签**展示，不是独立的条带：

```
[200 OK] [245ms] [1.2KB]                    [⌘\ 分屏] [⛶ 全屏]
  绿色     次要   次要                           工具按钮
```

- 状态码用语义色药丸
- 时间和大小用次要文字
- 工具按钮右对齐

## 6. 关键页面视觉规范

> 以下为像素级设计规范，开发时严格遵循。

### 6.1 侧边栏折叠态

折叠后仅显示图标列（52px）：

```
┌────┐
│ 🔍 │  ← 点击展开命令面板
├────┤
│ 📁 │  ← Collections 区域
│ 📜 │  ← History 区域  
│ 🔧 │  ← Environments 区域
├────┤
│ ⚙  │  ← 设置
└────┘
```

### 6.2 请求编辑器 Tab 的活跃指示条

```css
/* 底部指示条，跟随活跃 Tab 滑动 */
.tab-indicator {
  position: absolute;
  bottom: 0;
  height: 2px;
  background: var(--brand);
  border-radius: 1px;
  transition: left var(--duration-normal) var(--ease-smooth),
              width var(--duration-normal) var(--ease-smooth);
}
```

### 6.3 Tab 溢出处理

Tab 超出宽度时：
- Tab 最小宽度 100px，超出后文字截断 + 省略号
- 右侧出现 « » 滚动箭头
- 最右侧 ⋮ 按钮展开 Tab 列表下拉

### 6.4 拖拽分割线

```css
.split-handle {
  height: 4px;
  background: transparent;
  transition: background var(--duration-instant);
  cursor: row-resize;
}

.split-handle:hover {
  background: var(--brand);
}

.split-handle:active {
  background: var(--brand-hover);
}
```

### 6.5 变量高亮

URL 和输入框中的 `{{variable}}` 用品牌色高亮：

```css
.variable-highlight {
  color: var(--brand);
  background: var(--brand-muted);

.variable-highlight:hover::after {
  content: attr(data-value);
  position: absolute;
  /* tooltip 显示变量当前值 */
}
```
