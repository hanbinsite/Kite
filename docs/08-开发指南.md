# API Client 开发指南

> 本文档是设计文档到实际代码的桥梁，提供从零初始化项目的完整步骤、所有配置文件的完整内容、Rust 结构体定义、组件清单及错误码体系。

## 1. 项目初始化步骤

### 1.1 前置条件

```bash
# Node.js >= 20
node -v

# pnpm >= 9
npm install -g pnpm

# Rust >= 1.77
rustup update stable

# Tauri CLI
pnpm add -g @tauri-apps/cli
```

### 1.2 创建 pnpm workspace

```bash
mkdir api-client && cd api-client
pnpm init
```

创建 `pnpm-workspace.yaml`：

```yaml
packages:
  - "apps/*"
  - "packages/*"
```

### 1.3 初始化 Tauri 应用

```bash
pnpm create tauri-app apps/desktop --template react-ts
```

手动调整生成的目录结构，确保与 04-技术方案.md §2.2 一致。

### 1.4 设置 monorepo packages

```bash
# UI 组件库
mkdir -p packages/ui/src/components packages/ui/src/hooks packages/ui/src/contexts packages/ui/src/styles/tokens

# 核心业务逻辑
mkdir -p packages/core/src/http packages/core/src/graphql packages/core/src/grpc
mkdir -p packages/core/src/websocket packages/core/src/mqtt packages/core/src/sse
mkdir -p packages/core/src/collection packages/core/src/environment packages/core/src/script
mkdir -p packages/core/src/storage packages/core/src/codegen packages/core/src/importer packages/core/src/navigation

# 类型定义
mkdir -p packages/types/src

# 测试工具
mkdir -p packages/test-utils/src packages/test-utils/fixtures
```

### 1.5 安装 workspace 级依赖

```bash
# 构建工具
pnpm add -w -D turbo typescript @types/node

# 代码质量
pnpm add -w -D eslint prettier eslint-config-prettier @typescript-eslint/eslint-plugin @typescript-eslint/parser

# 根目录依赖
pnpm add -w react react-dom
pnpm add -w -D @types/react @types/react-dom
```

### 1.6 安装 desktop 应用依赖

```bash
cd apps/desktop

# 核心框架
pnpm add @tauri-apps/api @tauri-apps/plugin-shell @tauri-apps/plugin-fs @tauri-apps/plugin-dialog @tauri-apps/plugin-notification @tauri-apps/plugin-global-shortcut @tauri-apps/plugin-process

# UI
pnpm add @radix-ui/react-dialog @radix-ui/react-dropdown-menu @radix-ui/react-context-menu @radix-ui/react-tabs @radix-ui/react-toggle @radix-ui/react-tooltip @radix-ui/react-scroll-area @radix-ui/react-separator @radix-ui/react-avatar @radix-ui/react-select @radix-ui/react-slot @radix-ui/react-popover

# 样式与动画
pnpm add tailwindcss @tailwindcss/vite framer-motion

# 状态管理
pnpm add zustand immer

# 编辑器
pnpm add @monaco-editor/react @codemirror/lang-json @codemirror/lang-javascript @codemirror/lang-html @codemirror/lang-xml @codemirror/lang-css @codemirror/view @codemirror/state @codemirror/commands

# GraphQL
pnpm add graphql-request graphql

# 拖拽
pnpm add @dnd-kit/core @dnd-kit/sortable @dnd-kit/utilities

# 图标
pnpm add lucide-react

# 验证
pnpm add zod

# 国际化
pnpm add react-i18next i18next

# 虚拟滚动
pnpm add @tanstack/react-virtual

# 开发依赖
pnpm add -D vite @vitejs/plugin-react vitest @testing-library/react @testing-library/jest-dom jsdom
```

### 1.7 连接 workspace 依赖

在各子包 `package.json` 中使用 workspace 协议引用：

```json
{
  "dependencies": {
    "@api-client/types": "workspace:*",
    "@api-client/ui": "workspace:*",
    "@api-client/core": "workspace:*"
  }
}
```

### 1.8 验证开发环境

```bash
# 根目录执行
pnpm install

# 验证 TypeScript 编译
pnpm -r exec tsc --noEmit

# 启动 Tauri 开发模式
cd apps/desktop
pnpm tauri dev
```

impl From<reqwest::Error> for AppError {
    fn from(err: reqwest::Error) -> Self {
        let url = err.url().map(|u| u.to_string()).unwrap_or_default();
        if err.is_timeout() {
            AppError::NetTimeout { url, elapsed_ms: 0 }
        } else if err.is_connect() {
            AppError::NetConnectFailed {
                url,
                reason: err.to_string(),
            }
        } else if err.is_redirect() {
            AppError::NetTooManyRedirects { url, max: 10 }
        } else {
            AppError::NetSendFailed {
                url,
                reason: err.to_string(),
            }
        }
    }
}

impl From<rusqlite::Error> for AppError {
    fn from(err: rusqlite::Error) -> Self {
        AppError::StorageDbError {
            reason: err.to_string(),
        }
    }
}

impl From<rquickjs::Error> for AppError {
    fn from(err: rquickjs::Error) -> Self {
        AppError::ScriptExecutionFailed {
            message: err.to_string(),
        }
    }
}

impl From<std::io::Error> for AppError {
    fn from(err: std::io::Error) -> Self {
        AppError::StorageReadFailed {
            path: String::new(),
            reason: err.to_string(),
        }
    }

### 6.2 前端错误处理

```typescript
interface AppErrorResponse {
  code: string;
  detail: Record<string, unknown>;
}

function handleError(error: AppErrorResponse): void {
  const category = error.code.split("_")[0];
  const severity = getSeverity(error.code);

  if (category === "NET") {
    showNetworkError(error);
  } else if (category === "SCRIPT") {
    showScriptError(error);
  } else if (category === "VAULT") {
    showVaultError(error);
  } else {
    showGenericError(error);
  }

  if (severity === "critical") {
    logToSentry(error);
  }
}
```

## 7. TypeScript 类型同步策略

### 7.1 推荐方案：ts-rs 自动生成

使用 `ts-rs` crate 在 Rust 编译时自动生成 TypeScript 类型定义文件：

```rust
#[derive(Debug, Clone, Serialize, Deserialize, TS)]
#[ts(export_to = "HttpResponse.ts")]
pub struct HttpResponse {
    pub id: String,
    pub status: u16,
    // ...
}
```

编译后在 `src-tauri/generated/` 目录下生成 `.ts` 文件，通过 build script 自动复制到 `packages/types/src/generated/`。

### 7.2 构建集成

在 `src-tauri/build.rs` 中添加：

```rust
fn main() {
    tauri_build::build();
    // ts-rs 导出由 derive 宏自动完成
    // build script 结束后将 generated/ 复制到前端
    let out_dir = std::env::var("OUT_DIR").unwrap();
    let generated = std::path::Path::new(&out_dir).join("bindings");
    let dest = std::path::Path::new("../../packages/types/src/generated");
    if generated.exists() {
        let _ = std::fs::create_dir_all(dest);
        for entry in std::fs::read_dir(generated).unwrap() {
            let entry = entry.unwrap();
            let _ = std::fs::copy(entry.path(), dest.join(entry.file_name()));
        }
    }
}
```

### 7.4 备选方案

| 方案 | 优势 | 劣势 |
|------|------|------|
| `ts-rs`（推荐） | 轻量，derive 宏零侵入 | 不支持 OpenAPI |
| `specta` | 同时生成 TS + OpenAPI | 较重，API 变动多 |
| 手动同步 | 完全可控 | 维护成本高，容易遗漏 |
| `ts-rs` + CI 校验 | 自动化同步验证 | 需额外 CI 步骤 |

CI 校验脚本：

```yaml
- name: Verify type sync
  run: |
    cargo build
    diff -r src-tauri/generated/ packages/types/src/generated/ || exit 1
```

```typescript
// packages/test-utils/src/mock-invoke.ts
import { vi } from "vitest";

export function mockInvoke(handlers: Record<string, unknown>) {
  vi.mock("@tauri-apps/api/core", () => ({
    invoke: vi.fn((cmd: string, args?: Record<string, unknown>) => {
      const handler = handlers[cmd];
      if (typeof handler === "function") {
        return handler(args);
      }
      if (handler !== undefined) {
        return Promise.resolve(handler);
      }
      return Promise.reject(new Error(`Unknown command: ${cmd}`));
    }),
  }));
}

export function mockListen() {
  vi.mock("@tauri-apps/api/event", () => ({
    listen: vi.fn(() => Promise.resolve(vi.fn())),
    emit: vi.fn(),
  }));
}
```
